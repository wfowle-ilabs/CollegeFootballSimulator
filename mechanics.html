Mechanics — Checks, Ratings & the Inside Run

The M1 build spec

This page is the concrete spec for Milestone 1 (data model + PlayerGenerator + a single inside-run play). It pins the numbers the Architecture leaves abstract: the rating scale, the blend math, DC derivation, the inside-run check chain, and the first-pass curves. Curves here are deliberately first-pass and get tuned in M9 — the structure is what M1 commits to.

1 Ratings: scale & independent axes

Every rating — core attributes and position skills — lives on a 1–20 scale (BG3-pure). A Player carries two independent sets:

  • Core attributes (6): Strength, Agility, Speed, Awareness, Durability, Composure.
  • Position skills: the specific, checkable ratings for a position (for the inside run: OL RunBlock/AnchorStrength, DL BlockShed/RunStop, LB RunStop/Tackling, RB Vision/Elusiveness/Trucking/BallSecurity).

Why independent? Because checks blend a skill with an attribute (decision below), skills must not be derived from attributes — otherwise an attribute would count twice. The generator produces both axes with only a loose correlation (a fast RB tends to have higher Elusiveness, but they’re stored separately).

The 1–20 scale is intentionally coarse and dice-dominant. Granularity comes from the blend, not from a wide scale.

2 The check primitive

Every contested event is one d20 check (see the Resolution System for the philosophy):

roll   = d20
total  = roll + attackerModifier
DC     = 10 + defenderModifier          ← opposed check expressed as a static DC
success = roll == 20  OR  (roll != 1 AND total >= DC)
margin  = total - DC                     ← scales magnitude (e.g. yards)
crit    = roll == 20   (best-case, ignores DC)
blunder = roll == 1    (auto-fail, triggers a bad outcome)

Expressing the opposed check as DC = 10 + defenderMod keeps it legible and BG3-flavored: the defender is a passive difficulty, the attacker rolls against it.

2.1 The blend + modifier (the math)

A side’s modifier comes from blending the relevant skill with a supporting attribute, then applying the curve:

effectiveRating = w_skill · skill + w_attr · attribute       (both 1–20)
modifier        = floor((effectiveRating − 10) / 2)          (→ roughly −5..+5)

default weights : w_skill = 0.65 , w_attr = 0.35   (per-check tunable)

Blending the values before the curve (not the modifiers) is what produces many distinct effective ratings from a coarse 1–20 scale.

2.2 Advantage / disadvantage (the favorite lever)

When the two sides’ effective ratings are far apart, the stronger side rolls with advantage (2d20, take the higher); the weaker with disadvantage (take the lower):

if (attackerEffective − defenderEffective) >= ADV_GAP   → attacker rolls advantage
if (defenderEffective − attackerEffective) >= ADV_GAP   → attacker rolls disadvantage
ADV_GAP = 6   (first pass)

This is how a big talent gap makes a favorite reliably favored without widening the modifier range — the dice-dominant feel survives, but mismatches tilt correctly.

3 The inside-run recipe

One handoff up the middle resolves as a short chain of checks. Each link feeds the next; a decisive win or a crit can skip ahead.

3.1 1. The push (trenches)

The playside blocker vs the gap defender:

Attacker Defender
Skill OL RunBlock DL BlockShed / RunStop
Attribute Strength Strength
  • Success → a hole opens; margin sizes it (and the yards at the line).
  • Blunder (nat 1) → TFL: defender wins decisively, negative yards, chain ends.
  • Crit (nat 20) → clean lane; skip straight toward open field.

3.2 2. The second level

If the back reaches the linebackers:

Attacker Defender
Skill RB Elusiveness (or Trucking vs power) LB Tackling / RunStop
Attribute Agility Agility
  • Success → broken/avoided tackle, extra yards (YAC) from margin.
  • Failure → tackle made near the line of scrimmage.
  • Blunder or big hit at contact (ball-carrier nat 1, or a failure by a large margin) → trigger a fumble check: RB BallSecurity ⊕ Composure vs a fumble DC; failure = turnover.

3.3 3. Open field

Reached only via a decisive second-level win or a crit:

Attacker Defender
Skill / Attr RB Speed pursuit Speed
  • margin scales the breakaway length — a true long gain or TD.

4 Margin → yards (first pass)

A single transfer curve turns a check margin into yardage; the chain sums the contributions.

yardsAtLine ≈ clamp( round(2 + margin · 0.8), −3, 12 )
secondLevel ≈ clamp( round(margin · 1.0),      0, 20 )   (on success)
openField   ≈ clamp( round(8 + margin · 2.5),  0, 80 )   (breakaway only)
Push margin Result at the line
≤ −5 (or blunder) TFL, −2 to −3
−4 … 0 stuffed, 0–2
1 … 4 gain, 3–5
5 … 8 good push, 6–10
9+ (or crit) lane opens → second level

These constants are the M9 tuning surface. M1 commits to the shape (piecewise, margin-scaled, clamped), not the exact numbers.

5 Worked example

An inside run, traced end to end:

PUSH
  Playside Guard : RunBlock 14, Strength 15
                   eff = 0.65·14 + 0.35·15 = 14.35 → mod +2
  Interior DL    : BlockShed 12, Strength 13
                   eff = 0.65·12 + 0.35·13 = 12.35 → mod +1   → DC = 11
  gap = 2.0 (< 6) → no advantage
  roll d20 = 12 → total 14 vs DC 11 → SUCCESS, margin 3
  yardsAtLine = round(2 + 3·0.8) = 4   → hole opens

SECOND LEVEL
  RB  : Elusiveness 15, Agility 14 → eff 14.65 → mod +2
  LB  : Tackling 13,  Agility 12  → eff 12.65 → mod +1   → DC = 11
  roll d20 = 9 → total 11 vs DC 11 → SUCCESS, margin 0
  secondLevel = round(0·1.0) = 0   → tackle after a short gain

RESULT: +4 yards (no breakaway). A believable inside-run carry.

The point of the trace: every yard is explainable by the checks that produced it.

6 PlayerGenerator (basic)

Rosters are generated, deterministically, from the seeded RNG (same seed + team ⇒ same players):

  • Each attribute & skill ~ Normal(μ_position, σ ≈ 2–3), clamped 1–20. μ shifts with team prestige (a stronger program draws from higher means).
  • Axes stay independent: skills are only loosely correlated to their supporting attribute (enough to feel coherent, not derived).
  • Roster: generate enough to field the play — OL, interior DL, LBs, RB (a starters-plus set is fine for M1; full depth comes with M3).
  • Identity (stubbed): name from a pool, weighted class year, position-conventional jersey number.

Prestige-weighting and distribution shapes are first-pass; believable talent spreads are an M9 tuning target. The generator is also the seam recruiting replaces later — see player generation.

7 What this locks for M1

  • Ratings: 1–20, two independent axes (attributes + skills).
  • Check: d20 + attackerMod vs DC = 10 + defenderMod; crit/blunder on 20/1; margin scales magnitude.
  • Modifier: floor((0.65·skill + 0.35·attr − 10) / 2).
  • Big rating gaps → advantage/disadvantage (ADV_GAP = 6).
  • Inside run = push → second level → open field, with the margin→yards curves above.
  • Generator: seeded, per-position Normal, prestige-shifted, independent axes.

Everything numeric here is a tuning surface for M9; the structure is the M1 contract.
