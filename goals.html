v1 Goals & Milestones

College Football Simulator

Companion to the Architecture. Living document.

1 Vision

A single-player college football season simulator where the player picks an FBS team, sets their gameplan and depth chart, and simulates a full season — regular season, conference championships, and the complete CFP — to a national champion, then rolls forward through a lightweight offseason into the next season, year over year. Outcomes are driven by a Baldur’s Gate III-style character/attribute system resolved with d20-vs-DC dice. No traditional overall ratings.

The v1 north star: “Start a save, take a program, and run it season after season — believable, fully-simulated, year over year.”

2 v1 Definition of Done

v1 ships when a user can, end-to-end:

  1. Launch → Main Menu (New Save / Continue / Quit).
  2. Start a new save: choose a conference, then a team within it; confirm season/year.
  3. Land in the Season Hub: see schedule, roster, depth chart, standings, rankings.
  4. Configure the team: set a gameplan/scheme and arrange the depth chart (sim-and-watch — no live play-calling).
  5. Simulate games play-by-play, week by week (or sim-to-date), watching their team’s games resolve as a readable play log + live box score; other games resolve in the background.
  6. Progress through the season: standings, rankings, and stats update each week.
  7. Read the Media site: open the Media menu for weekly recaps, a Featured tab, and lookup of any team’s coverage (deterministic templated narratives in v1).
  8. Reach the postseason: conference championship games → full CFP bracket → bowls → national champion crowned.
  9. Roll into the next season: a lightweight offseason archives the year, advances class years, graduates seniors, and refills rosters with generated walk-ons — then a new season begins. Repeatable indefinitely.
  10. Save & resume at any point with deterministic continuation.

If all of these work for any FBS team in any conference, across multiple seasons, v1 is done.

3 Explicit Non-Goals for v1

(See the scope table for the full breakdown.) v1 does run multiple seasons, but the offseason replenishes rosters with generated walk-ons (player generation) rather than real recruiting. v1 does not include: real recruiting, transfer portal, coaching carousel, deep player progression, sub-FBS levels, live play-calling, 3D/2D field rendering, or any online/multiplayer features. These are deliberately deferred and the architecture leaves hooks for them.

4 Milestones (build order)

Each milestone is independently demonstrable and builds on the last.

4.1 M0 — Foundations

  • Godot 4.6.3 C# project scaffolding, folder structure, test harness.
  • CheckResolver (the d20-vs-DC primitive) + seeded RNG + unit tests.
  • Demo: run thousands of checks, confirm distributions match expectations.

4.2 M1 — Data Model & a Single Play

  • Core Resource model: Player, Team, Coach with the starting attribute set.
  • PlayerGenerator (basic): build a position-appropriate roster from the seeded RNG.
  • PlayResolver for one play type (e.g. inside run) composed from CheckResolver.
  • Demo: generate two teams, simulate one play between them; inspect the check trace.

4.3 M2 — Full Game Engine

  • Drive/Game simulators: clock, downs, field position, scoring, turnovers, punts/FGs, basic penalties.
  • PlayCaller (AI offense + defense) driven by coach attributes + situation.
  • StatAggregator → box score.
  • EventBus + typed domain events (GameConcluded, etc.) emitted by the engine — the backbone consumers subscribe to.
  • Demo: simulate a complete, believable single game with a full box score and play log.

4.4 M3 — League, Schedule & Season

  • League populated with all FBS teams/conferences; PlayerGenerator scaled to fill all ~130 rosters (prestige-weighted); ScheduleBuilder.
  • SeasonSimulator (week loop), StandingsService, RankingService, tiebreaks.
  • Demo: sim an entire regular season; standings & rankings look sane.

4.5 M4 — Postseason

  • ConferenceChampionships → CfpBracketBuilder (current 12-team format) → bowls → champion.
  • Demo: sim regular season straight through to a crowned national champion.

4.6 M5 — Save / Load & Storage

  • SaveManager + ISaveModule: thin spine (registry, RNG state, manifest) + per-module sidecars (current / season / media), by-ID references, atomic writes, schema versioning.
  • Demo: save mid-season, quit, reload, continue deterministically; confirm sidecars load independently.

4.7 M6 — Offseason & Multi-Season

  • OffseasonStage: archive completed season → historical-state sidecar; advance class years; graduate/depart seniors; PlayerGenerator walk-on replenishment; build the next season.
  • Multi-season loop wired through SeasonSimulator; historical records accumulate.
  • Demo: finish a season, run the offseason, play a second season; confirm history persists and rosters are refilled.

4.8 M7 — UI & Flow (the playable shell)

  • Main menu, new-save flow (conference→team), Season Hub, depth-chart/gameplan screens, game viewer (play log + live box score), postseason bracket view.
  • Demo: the full Definition-of-Done experience (Section 2), start to finish.

4.9 M8 — Media & Narratives

  • MediaSubsystem subscribes to game/week events; builds a NarrativeContext per story; tiered coverage (featured/ranked/upset/user = full; rest = templated recap).
  • TemplateNarrativeGenerator (deterministic baseline) → NewsArticles written to the sidecar media cache on a background thread between weeks.
  • Media menu UI: weekly recaps, Featured tab, team lookup.
  • Pluggable hook left ready for LlmNarrativeGenerator (LLamaSharp / GGUF) as a later drop-in.
  • Demo: advance a week, open the Media site, read auto-generated recaps and the Featured page; look up any team.

4.10 M9 — Balance & Polish

  • Tune modifier/DC/margin curves against believable stat lines (scoring, yards, completion %, etc.).
  • Tune PlayerGenerator distributions so rosters (and walk-ons) look like real CFB talent spreads.
  • Demo: a multi-season run whose aggregate stats pass a “does this look like real CFB?” sniff test.

M0–M4 are engine; M5–M6 add persistence and the season-over-season loop; M7 makes it playable; M8 adds the media layer; M9 makes it feel right. M1–M6 can largely proceed headless with tests before the UI exists.

5 Success Criteria (qualitative)

  • Believable: a simulated season’s leaders, scores, and upsets feel like real college football.
  • Legible: any outcome can be traced to the checks that produced it.
  • Deterministic: same save + seed reproduces the same season exactly.
  • Performant: simulating a full week (and a full season) is near-instant from the user’s perspective.
  • Extensible: adding FCS/D-II/JUCO later means adding data; real recruiting replaces walk-on replenishment in the existing offseason; new history/stat detail becomes a new sidecar — not an engine rewrite.

6 Beyond v1 (roadmap sketch)

  • v1.1 — SimCast & playstyle identity: a simulation broadcast of your team’s game (tabbed gamecast, drive-by-drive play-by-play, speed-controlled, real college clock, no 3D); a pre-game gameplan the AI caller honors (personnel, usage, reads, QB archetype, tempo, aggression); a real playbook of formations + concepts with offensive & defensive schemes and coach attributes wired in, so all ~130 teams play distinctly; real stadiums + simulated attendance that drives home-field advantage (and seeds the future ticket → NIL economy). Lays the IPlayCaller + Playcaller / Jump-In seam for the later 3D live play-calling mode. See the v1.1 design.
  • v2 — Recruiting & development: replace walk-on replenishment with a real recruiting cycle; richer player progression/aging in the (already-present) offseason.
  • v3 — Movement: transfer portal, coaching carousel, contracts.
  • v4 — Expansion downward: FCS, D-II, D-III, NAIA, JUCO/community colleges as additional leagues/levels.
  • Media v1.x — on-device LLM writer: swap the templated generator for a local ~7B/GGUF (LLamaSharp) writer behind INarrativeGenerator — richer, more varied articles with no 3rd-party API.
  • Later — Interaction & presentation: live play-calling / coach mode, richer game visualization; more event consumers (awards, achievements, recruiting buzz).
