Decisions Log

Locked decisions & open questions

The high-level decisions that anchor the Architecture and v1 Goals. Each entry is something we’ve committed to (or explicitly deferred), so we don’t re-litigate it later.

1 Locked decisions

Decision Choice Rationale
Engine Godot 4.6.3 Target engine version; API checked against Godot 4.6 docs.
Language C# / .NET (not GDScript) Data model = custom Resource types; sim logic = plain testable C# classes.
Sim granularity Play-by-play Every down simulated; richest stats and most legible outcomes.
Resolution system d20 + attribute modifier vs. Difficulty Class BG3-style. Nat 20 = crit, nat 1 = blunder. Uniform mechanic across all events.
Ratings Attribute & skill sheets — no “overall” Results emerge from the math; “ratings” are just a derived lens.
v1 league scope FBS (FBS-IAA) only All conferences/teams at one tier; lower levels are additive data later.
v1 lifecycle scope Multiple seasons (offseason = walk-on placeholder) Sim a season + full CFP → champion → lightweight offseason → next season, indefinitely.
Player generation Deterministic, prestige-weighted PlayerGenerator Builds initial rosters and offseason walk-ons; the seam recruiting plugs into later.
Offseason (v1) Archive → age roster → graduate → walk-on replenish The v1 stand-in for recruiting; keeps rosters whole across seasons.
User control Sim & watch Set scheme + depth chart pre-game; no live play-calling in v1.
Game speed Watch vs. quick-sim over one engine Non-player fixtures default to quick-sim; quick-sim discards the play log.
Determinism Single seeded RNG, saved with the game Same save + seed ⇒ identical season; enables replay/debugging.
World hierarchy Division → Conference → University → Athletics Dept → Team modeled in v1 Thin where unused, but present so the v2 economy bolts on without a rewrite.
Module communication Event-driven — EventBus + typed domain events Side effects via subscribers; the sim stays ignorant of its consumers.
Media subsystem In-game ESPN-style Media menu in v1 Immersion; a downstream consumer of deterministic game data.
Narrative generation Pluggable INarrativeGenerator; templated baseline in v1 LLM is a drop-in provider; de-risks v1 and keeps the Media menu working day one.
LLM runtime LLamaSharp + bundled GGUF, on-device (~7B) Fully self-contained — no 3rd-party API, no per-call cost.
Article coverage Tiered (featured/ranked/upset/user = full; rest templated) Quality-per-compute at ~65 games/week.
Article storage Sidecar media cache per save Persists & re-readable; keeps the core save lean.
Narrative determinism LLM/narratives strictly downstream, never feed the sim Preserves “same seed ⇒ same season.”
Storage model Thin spine + per-module sidecars (current/season/historical/media/stats-index) Keeps long multi-season runs lean; complexity grows by adding sidecars.
Cross-module references By ID, resolved via the spine registry — never object pointers Sidecars never point at each other or co-load; avoids coupling.
Cross-cutting data A dedicated stats-index, updated incrementally from events Leaderboards/records without loading every sidecar.

2 v1.1 — SimCast & playstyle identity

The v1.1 design commits to these (and refines two v1 calls above):

Decision Choice Rationale
Play Game options Playcaller (disabled) / SimCast / Quick Sim The game-day dropdown. Playcaller = the future 3D live-calling mode; SimCast = the v1.1 broadcast; Quick Sim = instant.
Game-day viewing SimCast — a simulation broadcast (no 3D) Watch your game paced + legible at a chosen speed; presentation over the same engine.
Game presentation A tabbed hub: Gamecast / Play-by-Play / Box Score / Team Stats / Game Info Modeled on an ESPN gamecast; drives are first-class objects under Play-by-Play.
User control (refined) Pre-game gameplan, AI caller honors it, watch it unfold Still “sim & watch” — a gameplan you authored, not a default. No live per-play input in SimCast.
Play timeline A typed GameTimeline (drives → segments); the string log is a derived view One deterministic source feeds every tab, the log, and the box score; speed is pure presentation.
Clock & pacing Real college clock: 1st-down-keeps-running (except last 2 min), 2-min timeout, tempo, timeouts, TV timeouts Resolves open question #3; produces the “pauses in the record.” NCAA rules, not NFL.
Play catalog (refined) A playbook of formations + concepts Resolves open question #2 toward richer concepts (not abstract buckets).
Schemes Offensive + defensive schemes as weighted playbooks Air Raid / Spread-RPO / Pro-Style / Smashmouth / Option vs. 4-3 / 3-4 / nickel / man-press / zone.
Coaching styles Wire existing Coach attributes into calling, clock, advantage A team’s identity = Scheme × Coach × Personnel × Gameplan.
Stadiums Each program gets a stadium (name / city / capacity) Data fill in FbsLeagueData; the venue the economy later hangs off.
Attendance & home-field Simulated attendance → home-field advantage in the dice First real HFA in the engine (crowd grants advantage); deterministic; the first driver of the future ticket → NIL economy.
Play-caller Realize IPlayCaller; one AiPlayCaller for all teams The documented seam; UserPlayCaller left for Playcaller / Jump-In.
Playcaller / Jump-In Present but disabled in v1.1 Two entry points to the future 3D live-calling mode — Playcaller (from Play Game, at kickoff) and Jump-In (inside SimCast, mid-game). The seam (interface + buttons) ships now; the mode does not.
Determinism (restated) Gameplan / scheme / coach / attendance are inputs; SimCast speed is presentation Same save + seed + gameplans ⇒ identical game; scrubbing speed can’t change a yard.

4 Open questions

Tracked design questions that don’t block the architecture but need answers as we build (see also the Architecture open questions):

  1. Exact attribute & skill list + the attr → modifier and margin → yards tuning curves (needs its own attributes reference).
  2. Play catalog granularity — resolved in v1.1: a real playbook of formations + concepts.
  3. Clock & game-flow fidelity — resolved in v1.1: real tempo / timeouts / TV timeouts / 2-minute model.
  4. Penalties & special-teams depth for v1.
  5. Data sourcing & generation — where real FBS team/conference identity data comes from, and the PlayerGenerator distributions (per-position, prestige-weighted) for believable rosters and walk-ons.
  6. CFP format specifics — exact bracket/seeding/bye rules and conference auto-bids.
  7. Quick-sim performance — whether full-engine quick-sim of a whole week is fast enough or needs an abstracted fast path.
  8. Local-LLM specifics — which ~7B model, quantization/size vs. quality, GGUF redistribution licensing, and the per-week background-generation time budget.
  9. Narrative “script” design — the structured prompt/template per article type and the guardrails that keep a small model constrained to NarrativeContext facts.
  10. Offseason depth & save growth — whether v1’s offseason applies any development/aging or strictly ages + walk-on replenishes, and expected sidecar (esp. historical) growth over many seasons.
