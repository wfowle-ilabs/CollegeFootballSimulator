College Football Simulator code is a .NET solution **`CollegeFootballSimGoDot.sln`** (classic format — Godot 4.6 generates/manages a `.sln`, NOT dotnet 10's default `.slnx`; created with `dotnet new sln --format sln`). Four projects:
- `src/CfbSim.Core/CfbSim.Core.csproj` — engine-free .NET class library (net8.0), **zero Godot dependency**. Holds the sim:
  - M1: `Rng` (`Pcg32Rng`/`IRng`), `Dice` (`DiceRoller`, `RollMode`, `D20Roll`), `Ratings` (`RatingMath`: blend + modifier), `Checks` (`CheckResolver`, `CheckResult`), `Model` (`Player`, `Team`, `Coach`, enums — **plain C# POCOs**, Resource/serialization deferred to M5), `Generation` (`PlayerGenerator`, `PositionProfiles`, `RosterSpec`, `NamePool`), `Sim.InsideRunResolver`.
  - M2: `Sim.Play` (`PlayOutcome`, `RunPlayResolver`, `PassPlayResolver`, `SpecialTeamsResolver`, enums), `Sim.PlayCalling.PlayCaller` (`PlaySituation`), `Sim.Game` (`GameSimulator`, `DriveSimulator`, `GameClock`, `Field`, `DriveResult`), `Stats` (`BoxScore`, `StatLines`, `StatAggregator`), `Events` (`EventBus`, `DomainEvent`, `GameConcluded`, `ScoreChanged`, `IEventSink`), `Model.GameResult`. Entry point: `GameSimulator.Simulate(rng, home, away, sink?, passBias?)`.
- `tests/CfbSim.Tests/CfbSim.Tests.csproj` — xUnit, headless. Run: `dotnet test tests/CfbSim.Tests/CfbSim.Tests.csproj`. (29 tests green at M2.)
- `samples/CfbSim.Demo/CfbSim.Demo.csproj` — console runner: simulates a full game + box score. **Arg passing:** `dotnet run --project samples/CfbSim.Demo -- <seed>` (the `--` is REQUIRED; run options otherwise swallow the arg) or run the built DLL directly: `dotnet samples/CfbSim.Demo/bin/Debug/net8.0/CfbSim.Demo.dll <seed>`.
- `CollegeFootballSimGoDot.csproj` (repo root) — Godot game project, `Godot.NET.Sdk/4.6.3`, net8.0. Uses `EnableDefaultCompileItems=false` + `Compile Include="src/Game/**/*.cs"` so it only compiles Godot-side code and references the core lib. Godot-side smoke test: `src/Game/SmokeTest.cs` (simulates a full game, prints to Output). Main scene `Main.tscn` (root Node + SmokeTest script) is set via `run/main_scene` in `project.godot`, so pressing Play in the Godot editor runs it.

Godot editor notes: open the project in the .NET build of Godot 4.6.3, let it build (Build button / on first run), press Play to see the smoke test in the Output panel. Core/sim development (M1-M6) needs no editor — it's all in `CfbSim.Core`, built/tested with `dotnet`. Editor matters at M7 (UI).

Naming gotcha avoided: static classes are `DiceRoller` and `RatingMath` (NOT `Dice`/`Ratings`) to avoid clashing with their own namespaces.

Determinism: everything random flows through `IRng` (seeded PCG32). Same seed + inputs ⇒ identical output. Don't use `System.Random` or Godot RNG in the core.

**M1 complete:** data model + `PlayerGenerator` + inside-run `PlayResolver` (push → second level → open field). Mechanics spec: `docs/mechanics.qmd`.

**M2 complete:** full play-by-play game engine — `GameSimulator` runs drives via `DriveSimulator` with clock/downs/field position/scoring/turnovers/punts/FGs/PAT/OT, an AI `PlayCaller` (offense play choice + defensive key), `StatAggregator`→`BoxScore`, and `GameConcluded` emitted on the `EventBus`. Playbook: inside/outside run, short/deep pass, punt, FG, XP. Kickoffs simplified to touchback-at-25; kickoff returns, 2-pt, timeouts/2-min logic deferred. Numeric constants (clock seconds, INT rates, yard curves) are first-pass — **full balance tuning is M9**. Produces believable finals (~19-11, 42-2) and box scores; ~160 plays/game.

**M3 complete:** league + season.
- `Model`: `League`, `Conference`, `Matchup`/`Schedule`, `SeriesRecord`. `Data`: `FbsLeagueData` (real ~135 FBS teams/conferences + prestige — alignment is ~2025, editable; Pac-12 is the 2-team post-collapse version), `RivalryData` (named protected rivalries). `Generation.LeagueBuilder`.
- `Services.SeriesHistory`: sparse head-to-head records (created on first meeting), named rivalries pre-seeded via `SeededFor(league)`; supports promoting a series to a rivalry.
- `Sim.Season`: `ScheduleBuilder` (per-conference round-robin via circle method + protected rivalries forced in + cross-conf non-conf fill + greedy week assignment, ~12 games/team over 14 weeks), `ConferenceScheduleRule`/`ScheduleRules` (divisions modeled but unused — hook for later), `SeasonSimulator` (week loop → `GameSimulator`, updates records + history, emits `WeekAdvanced`/`SeasonConcluded`), `StandingsService` + `TiebreakService`, `RankingService` (top-25, record + SOS-by-prestige + prestige prior — simplistic, improve later), `SeasonResult`/`TeamRecord`/`SeasonGameResult`/`RankedTeam`.
- Demo: `dotnet run --project samples/CfbSim.Demo -- league` (alignment), `-- season [seed]` (full season: top 25, P4 leaders, rivalry scores), `-- [seed]` (one game UGA vs ALA). 45 tests green.

**M4 complete:** postseason. `Sim.Postseason`: `ConferenceChampionships` (top-2 of each conference's standings play a title game; updates records + history), `CfpSelector` (12-team field: 5 highest-ranked conf champions auto-bid + 7 at-large, straight seeding 1–12), `BracketSimulator` (byes for seeds 1–4; First Round 8v9/5v12/7v10/6v11; Quarterfinal/Semifinal/National Championship; 11 games; higher seed nominal home, neutral, no HFA modeled), `BowlSelector` (light non-CFP bowl slate, 6+ win teams), `PostseasonSimulator` orchestrator, result types in `PostseasonResults.cs`. New event `NationalChampionCrowned`. Demo `-- season [seed]` runs season **and** postseason (conf champs, CFP field/bracket, champion, final Top 10). 51 tests green.

**Rule (per user):** EVERY game between two teams updates both head-to-head history AND the teams' season W/L records — regular season, conference championships, CFP, and bowls (the only future exception is the intra-squad spring game). Postseason carries real weight: `PostseasonSimulator` produces two polls — `SelectionRankings` (after champ games, seeds the CFP) and `FinalRankings` (after the whole postseason, with a playoff/bowl **achievement bonus** in `RankingService.Rank(..., achievementBonus)` so the champion finishes #1 and deep runs move the poll). Postseason W/L is applied centrally via `ApplyToRecords` (Wins/Losses/points; not ConfWins). Example: champion finishes 15-1 and #1.

The v1 Definition-of-Done core loop (start → season → champion) is now achievable headless.

**M5 complete:** Save/Load & Storage — **engine-free JSON** (System.Text.Json), NOT Godot ResourceSaver/.tres (keeps core testable headless; Godot will pass a globalized `user://` dir path later).
- `Save/SaveManager`: thin **spine** (`spine.json`: schemaVersion, year, userTeamId, RNG snapshot, sidecar manifest) + per-module **sidecars** (`league.json` current-state, `season.json` season-state, `history.json` head-to-head). Atomic writes (temp + `File.Move` overwrite). Schema version check throws `SaveManager.SaveVersionException`. `JsonSerializerOptions` uses `JsonStringEnumConverter` + **`PreferredObjectCreationHandling = Populate`** (REQUIRED — without it STJ skips our get-only collections like `Conferences`/`Teams`/`Roster`/`Schedule.Games` and they load empty). `Save/GameSave` is the in-memory aggregate.
- Resumable season: `SeasonState` (Schedule + Records + Games + NextWeek, persistable) + `SeasonDriver` (`Initialize`/`AdvanceWeek`/`ToResult`); `SeasonSimulator.Run` is now a thin wrapper over it (same behavior/determinism, M3/M4 unaffected).
- RNG resume: `Pcg32Rng.Snapshot()` → `Pcg32State` (state, inc, gaussian spare) and `Pcg32Rng.Restore(state)`.
- `SeriesHistory.Snapshot()`/`FromRecords()`. Computed get-only props across the model carry `[JsonIgnore]` (esp. `League.AllTeams`).
- Determinism on resume verified: save mid-season → reload → continue produces identical final records to an uninterrupted run. Demo `-- save`. 56 tests green. league.json ≈1.4MB (binary/compression is a later optimization). Per-sidecar (vs spine-level) version stamping deferred.

**M6 complete:** offseason & multi-season. `Sim.Offseason`: `OffseasonStage.Run` (archive completed season → `LeagueHistory`; advance class years; graduate seniors; refill each roster to `RosterSpec` with `PlayerGenerator.GenerateFreshman` walk-ons at team prestige) and `SeasonCycle` (`NewGame` builds league + primes id allocator; `RunFullYear` = schedule → season → postseason → offseason, advances `save.Year`). `Player.Class` is now settable (players age in place). `Model.LeagueHistory`/`SeasonSummary`/`SeasonFinish` = the season-by-season archive (champion, conf champions, all-team finishes). `GameSave` gained `Archive` (LeagueHistory) + `NextPlayerId`; **save schema bumped to v2** with new `archive.json` sidecar; spine carries `NextPlayerId`. Demo `-- multi [years]`. 61 tests green.

**v1 offseason design (resolves open Q#10):** NO attribute development in v1 — just age + graduate + walk-on replenish (deep progression is v2). Walk-ons generated at full team prestige (no decay) so the league stays stable until real recruiting (v2) adds talent variance.

**Known balance note (M9/v2):** with no recruiting/development variance, the highest-prestige team repeats as champion a lot (e.g., Georgia won 5 of 6 in one run). Parity comes later via recruiting (v2) and balance tuning (M9).

**M7 complete:** playable Godot UI shell. **Code-driven UI** (programmatic Godot Control nodes, no per-screen .tscn) so it builds/verifies via `dotnet build` (can't run the editor headless here; user verifies visually). `Main.tscn` root is now a `Control` with `src/Game/App.cs` (replaced SmokeTest.cs, which was deleted). Game-side code in `src/Game/`:
- `App.cs` — composition root + screen manager (swaps screens).
- `GameManager.cs` — flow/state controller (the SimDriver role): NewGame/SelectTeam/SimWeek/SimToEndOfSeason/RunPostseason/StartNextSeason/SaveGame/Continue + view helpers (CurrentRankings, UserConferenceStandings, UserResults). Save path = `ProjectSettings.GlobalizePath("user://saves/slot1")`.
- `UI/`: `Ui.cs` (control factory helpers), `MainMenuScreen` (New Game/Continue/Quit), `TeamSelectScreen` (conference→team), `SeasonHubScreen` (header record+week, Sim Week / Sim to End / Save / Main Menu, live conference standings + Top 25 + your results; "Go to Playoff" when season complete), `PostseasonScreen` (champion banner, CFP bracket, final Top 25, Start Next Season).
- Flow: Main Menu → Team Select → Season Hub (loop weeks) → Postseason → next season → Hub.
- **Godot game csproj now has `<ImplicitUsings>enable</ImplicitUsings>`** (it didn't before; UI code needs System/Linq/Generic). Screens are plain builder classes (not Node scripts) returning a Control tree — instantiated via `new`, no scene registration needed. No autoloads (folded into App/GameManager) to avoid project.godot churn; can promote later. Sim is synchronous (no background threading yet). 61 tests still green; whole solution builds.

To run: open in Godot editor (.NET build), Build, press Play.

**M8 complete:** media & narratives (event-driven). `CfbSim.Core/Media/`: `NewsArticle` (+ `ArticleType`), `NarrativeContext`, `INarrativeGenerator` (seam for future `LlmNarrativeGenerator` — LLamaSharp/GGUF, not built), `TemplateNarrativeGenerator` (deterministic templated text, phrasing varied by a stable hash — does NOT touch sim RNG, so determinism/resume preserved), `MediaStore` (per-save cache: ForWeek/Featured/ForTeam/WeeksWithCoverage), `MediaSubsystem` (builds `NarrativeContext` per game, **tiers** coverage → full article for user/rivalry/upset/both-ranked/top-10-winner, short recap otherwise; upset = prestige gap ≥12 or ranked-loses-to-unranked). Event-driven: `MediaSubsystem` consumes via `EventBus` — `GameManager` subscribes to `WeekAdvanced` and generates that week's coverage; `SeasonDriver.AdvanceWeek` is called with `_bus` from the game flow (headless season/multi tests pass no sink → no media, fine). Persisted: `GameSave.Media` + `media.json` sidecar; **save schema bumped to v3**. UI: `src/Game/UI/MediaScreen.cs` (week selector + team filter + headline list + reading pane), reached via a "Media" button on the Season Hub. Demo `-- media`. 66 tests green. Generation is synchronous (architecture wants off-thread between weeks — deferred); media kept for all weeks (retention/pruning deferred).

**Godot UI gotcha (recurring):** a code-built control only fills space if it has `SizeFlags.ExpandFill` AND every ancestor up the chain also does. Symptom of a miss: cards/panels render their title but the body collapses to zero height (looks "empty/broken"). `NavHost.Build` sets ExpandFill on its root for this reason; any new container that should fill must too.
